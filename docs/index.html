Reproducible TSDF Evaluation

Python utilities to measure runtime efficiency and 3D reconstruction quality for TSDF-based mapping methods. Used in the DB-TSDF paper and adapted from the metric-evaluation procedure introduced in SHINE-Mapping.

Repository Install with Docker See Metrics

Evaluation Code

The evaluation script loads predicted meshes (.ply, .pcd, .stl) and ground-truth point clouds, aligns them using multi-scale ICP, applies optional bounding-box cropping, and computes metrics such as Chamfer L1/L2, mean absolute error, precision, recall, and F-score.

Implementation details
  • Multi-scale ICP with voxel pyramid and point-to-plane estimation.
  • Optional ground-truth AABB cropping to reduce outliers from partial coverage.
  • Nearest-neighbor queries with KD-tree and uniform/Poisson sampling.

The pipeline is designed for large-scale LiDAR datasets and is fully reproducible. A Docker recipe with pinned dependencies is provided for consistent results.

Recommended practices
  • Set a fixed seed for random sampling and ordering.
  • Export metrics to CSV and log the evaluation configuration.
  • Visualize distributions (not only means) to capture variance.

Install using Docker

Clone the repository
git clone https://github.com/JoseMaese/mapping_metrics.git
cd mapping_metrics
Build the Docker image
docker build -t mapping_metrics .
Allow X server access (if GUI is required)
xhost +local:docker
Run the container
docker run -it \
  --env=DISPLAY \
  --env=QT_X11_NO_MITSHM=1 \
  --volume=/tmp/.X11-unix:/tmp/.X11-unix:rw \
  --name=mapping_metrics_container \
  mapping_metrics

The Dockerfile sets up the full Python environment and installs all dependencies automatically, allowing direct execution of the evaluation scripts inside the container.
